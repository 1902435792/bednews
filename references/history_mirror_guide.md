# 历史镜像使用指南

## 什么是历史镜像？

在《睡前消息》分析中，"历史镜像"指的是与当前议题具有**相似逻辑或结构**的历史案例。马督工经常使用这种方法来：
1. 避免重复论证（"我们在第X期讨论过类似问题"）
2. 展示问题的系统性（"这不是个案，是普遍现象"）
3. 预测发展趋势（"参考Y案例，我们可以预期..."）

---

## 强制使用要求

### Step 1 中必须执行

在 Bot 1 分析阶段，**必须**运行以下搜索：

```bash
# 搜索1：直接关键词
python .agent/resources/bedtime-news/scripts/search_index.py "[核心议题关键词]"

# 搜索2：底层逻辑/相关产业
python .agent/resources/bedtime-news/scripts/search_index.py "[产业类型] [制度类型]"
```

### 多维度搜索示例

| 议题 | 搜索1 | 搜索2 |
|------|-------|-------|
| 南京博物院文物争议 | `博物馆 文物 捐赠` | `国有资产 监管 腐败` |

## 如何使用搜索结果

### 1. 必须引用
在 `analysis_blueprint.md` 中，**必须**包含一个"历史对比"部分：

```markdown
## 历史镜像
| 历史案例 | 相似点 | 差异点 |
|----------|--------|--------|
| 第140期 独山县水司楼 | 都是地方政府资产管理失控 | 独山是增量问题，南博是存量问题 |
```

### 2. 可选引用
如果找到高度相关的历史节目，可以在文稿中直接引用：
> "我们在第X期讨论过类似的问题，当时的结论是..."

### 3. 如果没有找到
如果历史索引中没有相关案例，在蓝图中注明：
```markdown
## 历史镜像
无直接相关历史案例。这可能是一个值得深挖的"新议题"。
```

## 搜索技巧

### 关键词扩展
脚本已内置同义词扩展，例如：
- `文物` → 自动扩展为 `博物馆, 藏品, 古董, 文化遗产, 国宝`
- `腐败` → 自动扩展为 `贪腐, 贪污, 受贿, 行贿, 内部人`

### 禁用扩展
如果需要精确搜索，使用 `--no-expand` 参数：
```bash
python search_index.py "独山县" --no-expand
```

### 增加结果数量
默认返回5条，可调整：
```bash
python search_index.py "房地产 烂尾" --top 10
```

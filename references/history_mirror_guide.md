# 历史镜像使用指南

## v21.0 外部镜像补充

本地历史镜像检索优先级最高。只有在本地归档不足以形成参照时，再用外部搜索补充国际或跨地区案例：

```bash
python scripts/anysearch.py batch_search "[核心议题英文]" "[上位制度词英文] case study" "[国家] [类似问题] policy failure" --content-types news,doc,academic --zone intl --freshness year --max-results 4
```

拿到关键 URL 后必须读取正文：

```bash
python scripts/jina_reader.py "https://example.com/source" --max-chars 12000
```

外部案例只能作为“镜像参照”，不能覆盖本地往期结论；如果外部案例与本地归档冲突，必须说明制度环境、时间阶段、利益主体的差异。

## 什么是历史镜像？

在《睡前消息》分析中，"历史镜像"指的是与当前议题具有**相似逻辑或结构**的历史案例。马督工经常使用这种方法来：
1. 避免重复论证（"我们在第X期讨论过类似问题"）
2. 展示问题的系统性（"这不是个案，是普遍现象"）
3. 预测发展趋势（"参考Y案例，我们可以预期..."）

## 如何使用搜索结果

### 1. 必须引用
在 `analysis_blueprint.md` 中，**必须**包含一个"历史对比"部分：

```markdown
## 历史镜像
| 历史案例 | 相似点 | 差异点 |
|----------|--------|--------|
| 第140期 独山县水司楼 | 都是地方政府资产管理失控 | 独山是增量问题，南博是存量问题 |
```

### 2. 可选引用
如果找到高度相关的历史节目，可以在文稿中直接引用：
> "我们在第X期讨论过类似的问题，当时的结论是..."

### 3. 如果没有找到
如果历史索引中没有相关案例，在蓝图中注明：
```markdown
## 历史镜像
无直接相关历史案例。这可能是一个值得深挖的"新议题"。
```

## 搜索技巧

### 关键词扩展
脚本已内置同义词扩展，例如：
- `文物` → 自动扩展为 `博物馆, 藏品, 古董, 文化遗产, 国宝`
- `腐败` → 自动扩展为 `贪腐, 贪污, 受贿, 行贿, 内部人`

### 禁用扩展
如果需要精确搜索，使用 `--no-expand` 参数：
```bash
python search_index.py "独山县" --no-expand
```

### 增加结果数量
默认返回5条，可调整：
```bash
python search_index.py "房地产 烂尾" --top 10
```

## 💡 核心法则：历史承接与矛盾递进 (Historical Inheritance & Escalating Contradictions)

当历史镜像检索命中**高度相关**的往期节目时，新节目的蓝图和文稿撰写**严禁“平地起高楼”式的重新介绍**，必须遵循以下“历史承接”法则：

### 1. 继承既有结论，跳过基础科普
*   **做法**：如果以前节目已经科普过基础背景（如集采逻辑、分税制背景、医院编制体制），新文稿中**直接引用往期结论**，用一两句话带过科普，把篇幅节省出来探讨新矛盾。
*   **句式示例**：*“我们在第991期分析过，药店作为自负盈亏的民营主体，却要忍受地方医保极长的货款结算期，这使得高毛利非药成了它们唯一的自救灰色通道。今天出台的白名单新规，正是针对这一生存状态的最终清算。”*

### 2. 聚焦矛盾的“递进与演化” (Contradiction Escalation)
*   **分析视角**：不要平铺直叙地介绍新闻，而要将新闻定义为**“往期节目所揭示矛盾的最新升级版或变异版”**。
*   **分析轴线**：
    *   *过去*：博弈双方是地方执行弹性的“猫鼠游戏”（如事后检查、灰色缓冲）。
    *   *现在*：演化为算法穿透式监管下的“无摩擦清算”（如白名单硬编码网关）。
    *   *未来*：将导致微观网络崩溃，危机重新涌回体制大盘。

### 3. 构建“连续剧式”的智识链条 (Intellectual Seriality)
*   新文稿必须给观众一种**“原来这期节目是那期节目的第二季”**的智识愉悦感。在分析微观主体避险动作时，要对比他们在旧制度下的逃避方式与新制度下的变异方式，证明系统性规律在跨越时空后的持续有效性。
